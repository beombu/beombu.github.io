🌳 AST vs 🔗 Dependency Tracking

코드 분석의 두 가지 접근 방식

🌳 AST 분석

단일 파일 내부의 코드 구조를 파악합니다

  • 파일 1개만 분석
  • import/export 목록
  • 클래스 상속 구조
  • 함수 호출 패턴
  • 변수 선언 추적
  • 매우 빠름 (~100ms)

🔗 Dependency Tracking

파일들 간의 관계를 재귀적으로 추적합니다

  • 여러 파일 탐색 (최대 20개)
  • 의존 체인 구축
  • 순환 참조 감지
  • 영향 범위 분석
  • 재귀 깊이 제어 (최대 3)
  • 상대적으로 느림 (수초~수십초)

🏢 Java 개발자를 위한 비유

AST 분석 = javac의 파싱 단계
.java 파일 하나를 읽어서 클래스 구조, 메서드, 필드를 파악합니다.
IntelliJ의 "Structure" 창에서 보는 것과 비슷합니다.

Dependency Tracking = Maven/Gradle 의존성 그래프
pom.xml이나 build.gradle을 따라가며 전체 의존성 트리를 구축합니다.
mvn dependency:tree 명령어로 보는 것과 비슷합니다.

💻 코드 예시

// UserController.js (변경된 파일)
import UserService from './UserService';

export class UserController {
  async getUser(id) {
    return await this.userService.findUser(id);
  }
}

🎯 실전 사용 전략

📋 권장 설정

일반 MR (파일 1-10개): 
enable_ast_analysis: true   ← 항상 켜기
enable_dependency_tracking: false  ← 끄기 (불필요)
핵심 모듈 변경 (서비스 레이어, 공통 유틸): 
enable_ast_analysis: true
enable_dependency_tracking: true  ← 영향 범위 파악
급한 핫픽스: 
enable_ast_analysis: false  ← 빠른 배포 우선
enable_dependency_tracking: false
